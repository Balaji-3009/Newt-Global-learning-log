# GitHub Actions

Platform: Udemy  
Duration: May 27 – May 31

- Started the course by understanding CI/CD (Continuous Integration and Continuous Deployment) concepts and the role of GitHub Actions in DevOps workflows
- Learned GitHub Actions architecture and core workflow components
- Explored workflows, jobs, steps, actions, and runners
- Understood event-based workflow execution using triggers such as push, pull_request, workflow_dispatch, and schedules
- Learned YAML workflow structure and configuration basics
- Observed automation of build, test, and deployment workflows using GitHub Actions

## May 28

- Learned conditional execution using if expressions and workflow conditions

- Learned reusable workflows and workflow modularization techniques
- Understood workflow inputs, outputs, and data sharing between jobs
- Explored artifact upload and download workflows for build and deployment processes

## May 29
- Learned environment variables, secrets management, and secure workflow configuration
- Explored GitHub repository environments and deployment protection mechanisms

- Understood runtime configuration and environment-based workflow execution
- Explored workflow security considerations and best practices
- Learned cache management and dependency caching strategies to improve workflow performance

- Learned how jobs can run inside Docker containers for consistent execution environments
- Explored service containers and containerized testing workflows
- Studied integration of databases and external services within CI/CD pipelines
- Learned custom GitHub Actions development and reusable automation components
- Explored community actions and GitHub Marketplace integrations
- Observed real-world CI/CD pipeline implementation workflows using GitHub Actions
